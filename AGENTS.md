# Agent Repo Guidance

## 宪章

**完整宪章见 `docs/CONSTITUTION.md`(本仓库最高治理文档,与此摘要不一致时以宪章为准)。** 两条最常触发的原则:

1. **`docs/FUTURE.md`(未来特性登记册)中的任何内容不得进入实现。** 唯一例外路径:按该条目的「毕业标准」补齐证据 → 提升为核心规格(`docs/report_v2.md`)的正式条目 → 通过 spec-reviewer 评审。三步缺一不可(宪章 1.2)。
2. 核心规格只收录能写出「先失败的测试 + 通过条件」的内容;无 spec 不写实现,顺序恒为 spec → test → implementation(宪章 1.1 / 3.1)。新增内容不满足收录标准时,分流:有潜在价值的未验证想法 → `docs/FUTURE.md`;描述不足以裁定的 → 规格「待裁定项」;应舍弃的 → 记入变更摘要并说明理由。
